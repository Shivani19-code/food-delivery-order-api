Security

Authentication Flow

JWT-based auth with role-based access control.

üìù

Register

POST to /register with name, email, password. Password hashed with bcrypt.

üîë

Login

POST to /login ‚Üí receive a JWT token valid for 24 hours.

üîí

Authorize

Include Authorization: Bearer <token> header on protected routes.

üëë

Admin Access

Set user role: "admin" in DB to access /admin/** routes.

Getting Started

Quickstart

Run the API locally in 3 steps.

1 ¬∑ Clone & Run

git clone https://github.com/Shivani19-code/food-delivery-order-api.git cd food-delivery-order-api/backend go run ./cmd/api/

2 ¬∑ Register & Login

curl -X POST http://localhost:8080/register \ -H "Content-Type: application/json" \ -d '{"name":"Shivani","email":"s@test.com","password":"pass123"}' curl -X POST http://localhost:8080/login \ -H "Content-Type: application/json" \ -d '{"email":"s@test.com","password":"pass123"}'

3 ¬∑ Place an Order

curl -X POST http://localhost:8080/orders \ -H "Authorization: Bearer <YOUR_TOKEN>" \ -H "Content-Type: application/json" \ -d '{"restaurant_id":1,"order_items":[{"menu_item_id":1,"quantity":2}]}'

üå± Database is auto-seeded with Burger King & Pizza Hut on first run.
